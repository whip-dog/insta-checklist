# 📸 インスタ運用チェックリスト 2026

Instagramの開設から毎日の運用まで、これ1つで完結するチェックリストアプリ。
2026年の最新ルール（ハッシュタグ上限5個・オリジナル優遇・DMシェア重視など）に対応。

**公開URL:** https://whip-dog.github.io/insta-checklist/

🔒 **リベシティ会員限定**：合言葉（パスワード）でロックされている。合言葉を知っている人だけが中を開ける。
中身はAES-256で暗号化されており、合言葉なしではソースを見ても解読できない。合言葉は非公開（メモリに記録）。

チェック状態は各自の端末（localStorage）に保存される。一度合言葉を入れた端末は次回から自動で開く。

## 🔄 更新をお願いするとき（みわさん用）

インスタのルールはよく変わるので、変更があったら Claude にこう頼めば更新されます。テンプレ例：

> インスタのチェックリスト、〇〇のルールが変わったみたいなので最新に更新して。公開まで反映してほしい。

- 「公開まで反映して」と言えば、GitHub Pages への反映（下記の手順）まで自動でやります
- 具体的な変更点が分からなくても「最新ルールを調べ直して反映して」でOK。Claudeが裏取りしてから直します
- 反映後、URLは同じままで内容だけ新しくなります（1〜2分で反映）

## 🤖 Claudeへ：更新のデプロイ手順

このフォルダ（`insta-checklist/`）は **親の miyagi-dog リポジトリとは別の、専用リポジトリ**。
親からは `.gitignore` で除外済み。ここで独立して git 管理する。

- リモート: `https://github.com/whip-dog/insta-checklist`（PUBLIC）
- GitHub Pages: main ブランチ / ルート から配信
- 公開URL: https://whip-dog.github.io/insta-checklist/

**🔑 重要：アプリ本体は暗号化ゲートで保護されている。編集フローが2段階。**

- `index.src.html` … プレーンなアプリ本体（**編集はここを直す**）。`.gitignore`で公開repoに入れない＝ローカル専用
- `index.html` … `index.src.html`を合言葉で暗号化した公開用ファイル（**手で編集しない・自動生成**）
- `build-gate.js` … 暗号化ビルドスクリプト
- **合言葉はこの公開repoには書かない**。値は非公開メモリ `project_insta_checklist.md` に記録

更新手順（このフォルダ内で実行）:

```bash
# 1) アプリ本体を編集（index.html ではなく index.src.html を直す）
# 2) 合言葉で再暗号化して公開用 index.html を生成
node build-gate.js "＜合言葉＞"      # 合言葉はメモリ参照
# 3) 公開（index.src.html は.gitignoreで入らない）
git add -A
git commit -m "内容の要約"
git push
```

push すれば GitHub Pages が1〜2分で反映。ルール変更を反映したら `UPDATES.md` にも1行追記。
合言葉を変えたい時も `node build-gate.js "新しい合言葉"` → push でOK（メモリの合言葉も更新すること）。

**⚠️ 個人情報の注意:** このリポジトリは公開。`index.src.html`（平文）や親フォルダの `note-*.md` などは絶対にコミットしない（`.gitignore`で保護済み）。
